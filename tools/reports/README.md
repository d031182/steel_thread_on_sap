# Feng Shui Analysis Reports

This directory contains generated reports from Feng Shui analysis runs.

## Purpose

Centralized location for all Feng Shui quality analysis reports, including:
- Module-specific analysis reports (`feng_shui_report_[module_name].json`)
- Project-wide analysis reports (`feng_shui_report_all.json`)
- Text-based detailed reports (`.txt` files)

## Report Types

### JSON Reports
Generated by: `python -m tools.fengshui analyze --module [module_name]`

Contains structured analysis data including:
- Architecture violations
- Security issues
- UX/DX concerns
- Performance problems
- Test coverage gaps
- Documentation issues
- Module isolation violations

### Text Reports
Generated by: `python -m tools.fengshui analyze --detailed`

Human-readable format with:
- Detailed findings with code context
- Actionable recommendations
- Priority levels (CRITICAL, HIGH, MEDIUM, LOW)

## Usage

### Generate Report for Specific Module
```bash
python -m tools.fengshui analyze --module knowledge_graph_v2
```

### Generate Project-Wide Report
```bash
python -m tools.fengshui analyze
```

### Generate Detailed Text Report
```bash
python -m tools.fengshui analyze --detailed > tools/reports/feng_shui_detailed.txt
```

### Custom Output Location
```bash
python -m tools.fengshui analyze --output tools/reports/my_custom_report.json
```

## Git Tracking

⚠️ **Reports are NOT tracked in git** (see `.gitignore`)

Rationale:
- Reports are regenerated on demand
- Large JSON files would bloat repository
- Analysis results change frequently with code evolution
- Each developer generates their own local reports

## Best Practices

1. **Run before commits**: Generate reports to check code quality
2. **Use with Gu Wu**: Feng Shui reports feed into Gu Wu test gap analysis
3. **Archive important findings**: Extract key insights to `docs/knowledge/`
4. **Track progress**: Compare reports over time (locally, not in git)

## Related Tools

- **Shi Fu**: Session-start ecosystem analysis (`python -m tools.shifu --session-start`)
- **Gu Wu**: Test gap analysis and autofixing (`python -m tools.guwu analyze`)
- **Pre-commit**: Automated quality gates (`.pre-commit-config.yaml`)

## Report Lifecycle

1. **Generation**: On-demand via CLI
2. **Consumption**: Read by developers, Gu Wu, CI/CD
3. **Retention**: Local only (not committed)
4. **Cleanup**: Automatic via `.gitignore` exclusion
