Process Scenarios

Scenario 1: Material Procurement

Standard flow for purchasing materials:

  1. Create Purchase Order for materials
  2. Receive goods ‚Üí Create Goods Receipt
  3. Supplier sends invoice
  4. System performs three-way match (PO, GR, Invoice)
  5. Post invoice if no variances
  6. Execute payment per payment terms
Scenario 2: Service Procurement

Flow for purchasing services:

  1. Create Purchase Order for services
  2. Service is performed
  3. Create Service Entry Sheet ‚Üí Manager acceptance
  4. Supplier sends invoice
  5. System performs three-way match (PO, SES, Invoice)
  6. Post invoice and execute payment
Scenario 3: Variance Handling

When invoice differs from PO/GR:

  1. Invoice received with different price/quantity
  2. System detects variance automatically
  3. Invoice blocked with status = HELD
  4. AP team reviews variance reason
  5. Either: Reject invoice OR Approve variance
  6. If approved: Release block and process payment

Sample SQL Queries

üìå How to use: Copy these queries and run them in SQLite or your preferred database tool.
1. Complete P2P Cycle for a Specific PO

View the entire document flow from PO to Payment

SELECT * FROM vw_CompleteP2PTracking
WHERE PurchaseOrderID = 'PO-2024001';
2. Outstanding Invoices

Find all unpaid invoices with overdue tracking

SELECT * FROM vw_OutstandingInvoices
ORDER BY PaymentDueDate;
3. Invoices with Variances

Find blocked invoices with price/quantity mismatches

SELECT * FROM vw_InvoiceVariances
WHERE BlockingReason IS NOT NULL
ORDER BY InvoiceDate DESC;
4. Supplier Performance Report

Analyze supplier metrics and KPIs

SELECT * FROM vw_SupplierPerformance
ORDER BY TotalInvoiceAmount DESC;
5. Cash Discount Opportunities

Find invoices eligible for early payment discounts

SELECT
si.InvoiceID,
s.SupplierName,
si.GrossAmount,
si.CashDiscount1Amount,
CAST(julianday(si.CashDiscount1Date) - julianday('now') AS INTEGER) AS DaysRemaining
FROM SupplierInvoices si
JOIN Suppliers s ON si.SupplierID = s.SupplierID
WHERE si.PaymentStatus = 'UNPAID'
AND si.CashDiscount1Date >= date('now')
ORDER BY DaysRemaining;
6. PO Items Not Yet Received

Track outstanding deliveries

SELECT
poi.PurchaseOrderID,
poi.ItemNumber,
COALESCE(m.MaterialDescription, s.ServiceDescription) AS Item,
poi.OrderQuantity,
poi.QuantityReceived,
(poi.OrderQuantity - poi.QuantityReceived) AS Outstanding
FROM PurchaseOrderItems poi
LEFT JOIN Materials m ON poi.MaterialID = m.MaterialID
LEFT JOIN Services s ON poi.ServiceID = s.ServiceID
WHERE poi.QuantityReceived < poi.OrderQuantity;
